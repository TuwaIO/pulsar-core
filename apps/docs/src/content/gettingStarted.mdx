# Getting Started

This guide will walk you through the essential steps to integrate the **Pulsar** engine into your existing React application.

By the end of this guide, you will have a fully functional system for tracking and displaying user transactions across multiple blockchain ecosystems, including:

- A headless state management store powered by **`@tuwaio/pulsar-core`**.
- EVM chain support via **`@tuwaio/pulsar-evm`**.
- Solana support via **`@tuwaio/pulsar-solana`**.

Pulsar validates transaction metadata before wallet interaction or persistence. Transaction `title` strings are limited to 100 characters, `description` strings are limited to 300 characters, and serialized `payload` data is limited to 10KB. You can also add a `beforeTxProcess` callback in `createPulsarStore`, or override it per `executeTxAction`, to run project-specific preflight checks before any transaction starts.
